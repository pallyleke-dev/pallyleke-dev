The Brief

Building Information Modelling sits at the intersection of construction, engineering, and increasingly — AI. The task here was to take raw structural BIM models and apply semantic labels to every element so that AI systems could learn to identify structural components from 3D building data alone.

The models represented multi-storey structural frames — steel and concrete buildings with complex column grids, beam networks, floor slabs, and foundation elements. Each element needed to be correctly identified, labelled with the right IFC class, and documented with its structural role. The AI training pipeline depended on every label being consistent and accurate.

The Process

Step 01
IFC Model Import
Each structural model was imported into BlenderBIM/Bonsai from IFC format. The element tree was reviewed to understand the existing structure and identify unlabelled or mislabelled components.
Step 02
Element Classification
Every structural element was assigned its correct IFC class — IfcBeam, IfcColumn, IfcSlab, IfcWall, IfcFoundation — based on its structural function and geometry.
Step 03
Property Set Assignment
Structural properties were assigned to each labelled element: material, load-bearing status, fire rating, and dimensional data — all embedded as IFC property sets.
Step 04
Spatial Hierarchy Review
IFC models require elements to sit within a spatial hierarchy — Site → Building → Floor → Element. Every annotated element was verified to sit in the correct spatial container.
Step 05
Clash Detection Check
Elements were checked for spatial clashes — columns intersecting slabs unexpectedly, beams misaligned to the structural grid — and corrected before final annotation export.
Step 06
Export & Validation
Final IFC files were exported and validated against the IFC schema using an IFC checker to confirm all labels, properties, and hierarchies were correctly formed.

Challenges & Solutions

Non-standard element names. Many imported IFC files used generic names like "Element 001" with no structural context. Each had to be assessed geometrically and structurally before a label could be assigned — a slow but necessary process.

Mixed structural systems. Some models combined steel and concrete elements within the same frame. The annotation schema had to accommodate material differences without breaking the IFC class structure.

IFC validation errors. Initial exports occasionally failed IFC schema validation due to missing required properties. A validation-first workflow was adopted — check before export, not after.
