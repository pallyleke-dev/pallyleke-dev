The Brief

Object detection is one of the foundational tasks in computer vision — the ability for an AI to look at an image and identify what's in it, where it is, and what class it belongs to. But before any model can do that, a human has to teach it. That teaching happens through annotation.

This project involved preparing large-scale bounding box annotation datasets for object detection models. The images contained industrial components, mechanical parts, and assembly elements — objects the model needed to learn to detect and classify reliably. Every annotation had to be precise: a box even slightly too large or too small changes what the model learns.

The Process

Step 01
Image Batch Review
Each image batch was reviewed before annotation — checking image quality, lighting consistency, and object visibility. Blurred or unusable images were flagged and removed from the batch.
Step 02
Class Schema Setup
The class taxonomy was defined per project — what object types to detect, how to handle overlapping objects, and what minimum confidence threshold to apply when an object was partially visible.
Step 03
Bounding Box Drawing
For each object in each image, a tight bounding box was drawn — as close to the object boundary as possible without clipping. Box coordinates were recorded in [x, y, width, height] format.
Step 04
Class Label Assignment
Each box was assigned its class label from the taxonomy. Objects with ambiguous class membership were flagged for review rather than guessed — accuracy over speed.
Step 05
Confidence Scoring
A confidence score was assigned to each annotation — reflecting how clearly visible and unambiguously classifiable the object was. Low-confidence annotations were reviewed by a second pass.
Step 06
QA & Dataset Export
The complete annotated dataset was reviewed — checking for missed objects, incorrect boxes, and class inconsistencies — before export in the required format for model training.

Challenges & Solutions

Occluded objects. Many images contained partially hidden objects — a component behind another, or cropped at the image edge. A clear occlusion policy was established: annotate if more than 50% of the object is visible, flag the occlusion percentage in the metadata.

Class ambiguity. Some components looked similar enough that class assignment was genuinely uncertain. Rather than guessing, these were logged in an ambiguity register and resolved by reviewing the class definitions against the object geometry carefully before deciding.

Box tightness consistency. Different annotators draw boxes differently — some too tight, some too loose. A calibration exercise established a shared standard, and random samples were reviewed across batches to maintain consistency.
