The Brief

Point cloud annotation sits at the frontier of AI training data work. Unlike images — which are flat and familiar — point clouds are raw 3D spatial data: millions of individual points captured by LiDAR or structured light sensors, each with an x, y, z coordinate and often an intensity value. There are no lines, no edges, no clean shapes. Just points in space.

The task was to identify meaningful object clusters within these point clouds, assign class labels to each cluster, and document spatial boundaries precisely enough for autonomous perception models to learn from the data. This required strong 3D spatial reasoning and a methodical approach to what is, visually, a dense and chaotic dataset.

The Process

Step 01
Scan File Loading
Raw .pcd and .las scan files were loaded and visualised in 3D. Initial inspection assessed point density, scan coverage, and the presence of noise or artefacts requiring pre-processing.
Step 02
Noise Filtering
Statistical outlier removal was applied to clean the point cloud — eliminating stray points far from the main cluster density that would otherwise contaminate the annotation.
Step 03
Cluster Segmentation
Point clusters were segmented — isolating spatially distinct groups of points that corresponded to individual objects. Density-based segmentation was the primary method, supplemented by manual review for ambiguous regions.
Step 04
Class Label Assignment
Each identified cluster was assigned its class label based on its geometric profile and spatial context. Labels followed the project class schema — no improvised classes, no guesses.
Step 05
Boundary Documentation
Tight 3D bounding volumes were fitted around each labelled cluster — axis-aligned boxes for regular objects, oriented bounding boxes for objects at angles. Boundary coordinates were recorded per cluster.
Step 06
QA Review & Export
Annotated scans were reviewed by rotating through multiple viewpoints — checking for missed clusters, label errors, and boundary fit quality — before final export.

Challenges & Solutions

Cluster ambiguity. In dense scans, two nearby objects could have overlapping point regions with no clear boundary. A minimum gap threshold was established — if objects were closer than the threshold, each point was assigned probabilistically based on distance to the nearest cluster centroid.

Noise vs. real data. Not all stray points are noise — some are real but sparse parts of the scene. A conservative noise filter was used, retaining borderline points for manual review rather than automatically deleting anything that looked isolated.

Viewpoint dependency. Point cloud annotation looks different from every angle. Each annotated scan was reviewed from at minimum four viewpoints — top, front, side, and isometric — before sign-off.
